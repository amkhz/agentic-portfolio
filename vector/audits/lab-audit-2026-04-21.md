# Speculative Design Lab — Technical Audit

**Date:** 2026-04-21
**Scope:** `src/lab/`, `core/lab/`, `design-system/lab-tokens.css`, seven guides under `core/lab/guides/`
**Out of scope:** portfolio UI at `src/` and `core/content/`
**Dev URL:** http://localhost:5173/labs.html

## Anti-Patterns Verdict

**Pass, with caveats.** The lab does not read as AI-generated. Specific reasons it avoids the 2024–2025 fingerprints:

- Fonts are distinctive (Podkova + Georgia + JetBrains Mono), not Inter / Roboto / system defaults.
- Backgrounds are tinted (`oklch(0.08 0.003 270)`), not pure black.
- Layout is left-aligned and asymmetric, not centered hero templates.
- Typography has rhythm — three distinct sizes in the heading scale, deliberate tracking on mono labels.
- No gradient text. No glassmorphism-as-decoration (the one use — sticky nav with `bg-lab-bg-deep/90 backdrop-blur` — is functional, not ornamental).
- Accent glow on guide cards is restrained (single 10px blur, color-mixed to 60%), not neon-on-dark.
- Status pills are simple rounded rectangles with earned hierarchy (complete > in-progress > draft), not decorative.

**Caveats worth naming, not scoring as failures:**

1. **Dark mode with accent glow.** The lab runs dark-first with colored accents. Justified by VECTOR.md direction ("Blade Runner + Finn Juhl") and ADR-009, but watch that future additions don't escalate the glow vocabulary.
2. **Card grid on library index.** Three-column identical cards with kicker + title + author + status. Functional for a library, but if the library grows past ~20 guides, consider varying card emphasis (featured / active / archived).
3. **Heavy monospace usage.** JetBrains Mono carries kickers, meta, status pills, counters, section-nav pills, figure captions. Academic-preprint-viewer context warrants it, but it's close to "lazy technical vibe" territory. Watch the ratio as the lab evolves.

## Executive Summary

- **Total issues:** 14 (1 P0 critical, 3 P1 high, 5 P2 medium, 5 P3 low)
- **Top 3 most critical:**
  1. **P0 — Four DIRD guides have an accent hex of `#0d1620`, 1.09:1 contrast** against lab background. Migration-script artifact. Makes glossary terms, section-nav pills, tab underline, focus outlines, and card hover states effectively invisible on those four guides. Fails WCAG 1.4.3 and 1.4.11.
  2. **P1 — Glossary view skips h1 → h3.** Letter headings in `GuideGlossaryView` are h3 but the parent page only has one h1 (guide title) and no intervening h2 while the glossary tab is active. Violates WCAG 1.3.1.
  3. **P1 — `GuideDefinitionCard` animates `height`.** Motion/react height-to-auto expansion triggers layout on every frame. Works fine for one card but costs performance if multiple definitions are open. Anti-pattern per frontend-design DON'Ts.
- **Overall quality:** **B-.** Solid architecture and a11y baseline (skip links, landmarks, ARIA tab pattern, roving tabindex, reduced-motion respected throughout). One content-pipeline defect produces a visible regression on 4/7 guides. Fix the accent issue and this jumps to an A-.
- **Recommended next steps:**
  1. Ship P0 fix first (edit the four DIRD accent frontmatter values — ~2 minute edit).
  2. Fix P1 heading hierarchy in glossary view, swap `GuideDefinitionCard` height animation to transform/opacity.
  3. Proceed to `/polish` for P2/P3 refinements.

## Detailed Findings by Severity

### P0 — Critical

#### P0.1 — DIRD guides have an invisible accent color

- **Location:** `core/lab/guides/dird-13-warp-drive.md:9`, `dird-14-superconductors-gravity.md:9`, `dird-15-vacuum-spacetime-engineering.md:9`, `dird-28-breakthrough-cockpits.md:9`
- **Severity:** Critical (P0)
- **Category:** Accessibility + Content
- **Description:** All four DIRD guides have `accent: "#0d1620"` in frontmatter. Measured contrast against `--lab-bg-deep` is **1.09:1** — effectively invisible.
- **Impact:** On those four guides, *every* interactive element that uses `--guide-accent` for color or border becomes near-invisible:
  - Glossary `|term|` underlines and hover state (`GuideTerm.tsx`)
  - Tab bar active underline (`GuideTabBar.tsx`)
  - Section-nav active pill border and text (`GuideSectionNav.tsx`)
  - Guide card hover border on the library index (`GuideCard.tsx`)
  - Prev/Next hover border (`GuidePrevNext.tsx`)
  - Global focus outline from `src/lab/styles/lab.css:skip-link` and `a:focus-visible` rules
- **WCAG:** 1.4.3 Contrast (Minimum), 1.4.11 Non-text Contrast, 2.4.7 Focus Visible
- **Root cause:** The T10 migration script's "first non-neutral hex" heuristic picked the DIRD page background (`#0d1620`) instead of the actual accent because that value was absent from the hard-coded neutrals list. The real per-DIRD accents are:
  - `dird-13-warp-drive`: `#f0a050` (amber, 9.28:1)
  - `dird-14-superconductors-gravity`: `#e8a849` (warm amber)
  - `dird-15-vacuum-spacetime-engineering`: `#b48efa` (violet)
  - `dird-28-breakthrough-cockpits`: `#4ecdc4` (teal, 10.22:1)
- **Recommendation:** Edit each DIRD's frontmatter `accent:` line to the real value. Validate via `npm run audit:orphans`-style probe (or just re-run build + eyeball a guide page). Optionally harden the migration script by filtering hexes whose luminance is under 0.05 (near-black backgrounds) before accepting them as accent candidates.
- **Suggested command:** `/normalize` (align accents to the token system) or a targeted manual edit since scope is 4 lines.

### P1 — High

#### P1.1 — Heading hierarchy skips h2 in Glossary view

- **Location:** `src/lab/components/guide/GuideGlossaryView.tsx:70`
- **Severity:** High (P1)
- **Category:** Accessibility (semantic HTML)
- **Description:** When a guide is in Glossary mode, the page contains one h1 (`GuideHeader`) and then letter-group headings at h3. There's no h2 between them.
- **Impact:** Screen reader users jumping by heading level won't find these letter landmarks. Outline tools will flag the skip.
- **WCAG:** 1.3.1 Info and Relationships, 2.4.10 Section Headings (AAA target).
- **Recommendation:** Promote the letter-group headings to h2, or wrap the whole glossary view in a single h2 ("Glossary") with letter groups as h3 under it. The second reading is stronger — it gives the glossary itself a landmark.
- **Suggested command:** targeted fix (small scope).

#### P1.2 — `GuideDefinitionCard` animates `height`

- **Location:** `src/lab/components/guide/GuideDefinitionCard.tsx:20-27`
- **Severity:** High (P1)
- **Category:** Performance / Anti-pattern
- **Description:** The motion/react animation transitions `height: 0` → `height: "auto"`. This triggers full layout on every animation frame. Acceptable with one open card but compounds if the reader opens several (which the `GuideParagraph` state machine encourages via its multi-open design).
- **Impact:** Potential jank on older devices, especially mid-guide when multiple definitions are open and the user scrolls. Fails the frontend-design DON'T: "Animate layout properties (width, height, padding, margin) — use transform and opacity only."
- **Recommendation:** Replace the height animation with an opacity + small `translateY` crossfade. The expanded card then shows its final size immediately — acceptable trade for no layout thrash. Alternatively, use `motion`'s `layout` prop which measures once per toggle.
- **Suggested command:** `/optimize` or targeted fix.

#### P1.3 — `GuideTerm` has no `aria-controls` pointing to the definition card

- **Location:** `src/lab/components/guide/GuideTerm.tsx:10`, `GuideDefinitionCard.tsx:19`
- **Severity:** High (P1)
- **Category:** Accessibility (ARIA)
- **Description:** `GuideTerm` is a `<button aria-expanded>` but doesn't reference the card it expands via `aria-controls`. The card itself has `aria-label` but no id.
- **Impact:** Assistive tech can't announce "press Enter to expand definition; definition is in group X." Works today because the card renders immediately after the term's paragraph — proximity helps screen readers — but the formal link is missing.
- **WCAG:** 4.1.2 Name, Role, Value (relaxed interpretation) + WAI-ARIA Authoring Practices for Disclosure pattern.
- **Recommendation:** Give the card a deterministic id (e.g., `def-${term-slug}`), pass it down to `GuideTerm` as `aria-controls`. `GuideTerm` needs a local way to produce a consistent slug from the term string; trivial.
- **Suggested command:** `/harden` or targeted fix.

### P2 — Medium

#### P2.1 — Several touch targets below WCAG AAA 44×44

- **Location:** `GuideTabBar.tsx` (~40px tall), `GuideSectionNav.tsx` pills (~28px tall), `GuideGlossaryView.tsx` letter buttons (32×32), `GuideDefinitionCard.tsx` close-× (~24×24)
- **Severity:** Medium (P2)
- **Category:** Responsive / Accessibility
- **Description:** All meet the WCAG AA 2.5.8 minimum (24×24 CSS px) but fall short of the AAA 2.5.5 44×44 target.
- **Impact:** Slightly harder for users with motor impairments on touch devices.
- **Recommendation:** Bump tab `pb-3` to `py-3.5`, section-nav pills `py-1.5` to `py-2.5`, glossary letters from h-8/w-8 to h-11/w-11, close-× from `p-1` to `p-2`. Small visual change, real tap-target gain.
- **Suggested command:** `/adapt` or targeted fix.

#### P2.2 — Guide card accent dot uses a shadow glow

- **Location:** `src/lab/components/library/GuideCard.tsx:40`
- **Severity:** Medium (P2)
- **Category:** Anti-pattern sensitivity
- **Description:** `shadow-[0_0_10px_color-mix(in_oklab,var(--guide-accent)_60%,transparent)]` on the territory dot. Flirts with the "dark mode with glowing accents" fingerprint.
- **Impact:** Ambient rather than screaming, but if the library index adds more accent glows later (active-guide indicator, "new" pip), the page could tip into AI-slop territory.
- **Recommendation:** Keep it for now — it's restrained and it *does* help the dot read at the card's scale. Set a ceiling: no additional glows on the library index without dropping this one. Document in the polish pass notes.
- **Suggested command:** `/quieter` if Justin wants to dial back.

#### P2.3 — Kicker inconsistency across guides

- **Location:** `core/lab/guides/*.md` frontmatter `kicker:` field
- **Severity:** Medium (P2)
- **Category:** Content coherence
- **Description:** `emergent-quantization`: "Paper Explainer" (migrated from JSX source). DIRDs: "DIRD Guide Series". `quantum-gravity`: "Synthesis Guide". `uapx-field-methods`: "Research Guide Series". Feels haphazard across cards on the library index.
- **Impact:** Library index reads as inconsistent publication series. Low user impact but weakens editorial voice.
- **Recommendation:** Decide on a small vocabulary (e.g., "DIRD Guide Series", "Research Guide Series", "Synthesis Guide") and normalize. "Paper Explainer" → "Research Guide Series" is the obvious candidate for emergent-quantization.
- **Suggested command:** `/clarify` or Writer follow-up.

#### P2.4 — Em-dashes in guide copy conflict with VECTOR.md constraint

- **Location:** Multiple guide bodies (source authors' original prose), plus `GuideHeader.tsx:30` (source-line template uses ` · ` dividers).
- **Severity:** Medium (P2)
- **Category:** Content / style guide
- **Description:** VECTOR.md hard constraint: "No em-dashes in copy." Most guides inherit em-dashes from their source authors (legitimate reproduction), but the description fields are Tyrell-authored and some contain them.
- **Impact:** Violates a declared project constraint. Inconsistent with the portfolio side.
- **Recommendation:** Make a carve-out explicit: VECTOR.md says "no em-dashes in copy Tyrell writes"; ingested external content keeps its authors' punctuation. Or normalize descriptions. Writer's call.
- **Suggested command:** `/clarify` or explicit VECTOR.md amendment.

#### P2.5 — Section-nav sticky shadow is absent — active pill can feel "floating"

- **Location:** `GuideSectionNav.tsx:39`
- **Severity:** Medium (P2)
- **Category:** Visual polish
- **Description:** The sticky nav uses `border-b border-lab-border-subtle bg-lab-bg-deep/90 backdrop-blur` but no shadow. Once the page scrolls under it, the divider is the only boundary cue. Works, but the nav can feel like it's floating in the content on certain transitions.
- **Impact:** Minor legibility at section-boundary moments.
- **Recommendation:** Add a subtle shadow (`shadow-[0_1px_0_var(--lab-border-subtle)]` or similar) that strengthens when scroll position > top.
- **Suggested command:** `/polish`.

### P3 — Low

#### P3.1 — No `lang` attribute on individual guide content sections
Currently the entire lab is `<html lang="en">`. Guide quotes/terms in non-English (German, Greek letters, math symbols) don't use `lang="..."` at the span level. Screen-reader pronunciation may drift. Minor.

#### P3.2 — GuideFigure has no explicit dimensions
`<img>` loads lazy but doesn't reserve space. On slow connections, figures push content down as they resolve. Add `width` + `height` attributes if feasible, or intrinsic aspect ratio via CSS. Minor layout-shift cost.

#### P3.3 — `GuideGlossaryView` sort collapses terms starting with non-letters
`groupLetter` returns `"#"` for anything whose first character isn't `[a-zA-Z]`. Greek-letter terms (σ, ω) land in `#`. Works, but the bucket will look orphaned if used. None of today's guides hit this, so P3. Flag for when math-heavy guides arrive.

#### P3.4 — `Helmet` meta description in `GuideView` concatenates description + "Source: ..."
Some guides have long descriptions; the final meta description can exceed 160 chars. Search engines will truncate. Trim or split.

#### P3.5 — `labs.html` `data-lab="dark"` attribute unused
Added to mirror portfolio's `data-theme="dark"` but the lab has no theme toggle. Harmless but dead. Remove or wire up.

## Patterns & Systemic Issues

1. **Migration-pipeline trust boundary.** The T10 script's heuristic accent extractor surfaced a blind spot: near-black hexes outside the explicit neutrals list become false-positive accents. Four guides affected. Follow-up: harden the script to reject hex values whose luminance is under a threshold (e.g., 0.05) before accepting them. Alternatively: require `--accent` as an explicit CLI flag so authors can't inherit an auto-detected wrong color silently.

2. **Heading-hierarchy pattern in glossary view.** The one skip in `GuideGlossaryView` is small, but it's the only place in the lab tree where the hierarchy isn't provably correct. Worth a short lint rule or structural test to prevent regressions as more tab-panel views ship.

3. **Reduced-motion is respected everywhere.** Four components (`GuideSectionNav`, `GuidePrevNext`, `GuideGlossaryView`, `GuideDefinitionCard`) independently check `prefers-reduced-motion`. Good coverage, slight duplication — could extract to a single `useReducedMotion` hook once the lab grows. Not urgent.

4. **Per-guide accent propagation via CSS custom property** works cleanly. `GuideRenderer` and `GuideCard` both set `--guide-accent` via inline style (documented ADR-009 exception). Everything downstream uses the cascade. No raw hex leaks into lab UI code.

## Positive Findings

- **Skip link + semantic landmarks.** `LabLayout` provides a working skip-to-content link; every page has `<header>`, `<main id="main">`, `<footer>` landmarks.
- **ARIA tab pattern is complete.** `GuideTabBar` implements the full WAI-ARIA disclosure pattern — roving tabindex, arrow-key navigation, aria-selected, aria-controls. Rare to see done properly.
- **Reduced-motion coverage.** Every component that moves respects `prefers-reduced-motion: reduce` via explicit checks, plus a global `@media` rule in `lab.css` as a belt-and-suspenders fallback.
- **Guide description meta wiring.** `GuideView` sets `<title>`, `<meta name="description">`, Open Graph title/type. Shareable URLs will produce correct previews on day one.
- **External links have `rel="noopener noreferrer"`.** Source-citation `<a>` tag and footer GitHub link both scoped correctly.
- **No inline styles in lab UI except the two documented exceptions.** `GuideRenderer` and `GuideCard` both use the `--guide-accent` CSS variable pattern explicitly documented in ADR-009. Everything else flows through tokens.
- **Non-lossy figure container.** The off-white `--lab-figure-bg` paper container is the right call for mixed photographic + diagrammatic figures. UAPx figures with white backgrounds look natural; line-art diagrams stay legible.
- **Accessible text contrast across the palette.** `--lab-text-primary` (16:1), `--lab-text-secondary` (~9:1), `--lab-text-muted` (~4.8:1) all clear AA against `--lab-bg-deep`.
